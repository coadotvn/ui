Block Grids

Block grids give you a way to evenly split contents of a list within the grid. If you wanted to create a row of five images or paragraphs that need to stay evenly spaced no matter the screen size, the block grid is for you.

Basic

Use a simplebgsm-#classes to code up the block grid and specify the number of items in a row.

If you use thesm-#only, the grid will keep its spacing and configuration no matter the screen size. If you uselg-#only, the list items will stack on top of each other for small devices. If you use both of those classes combined, you can control the configuration and layout separately for each breakpoint.

Customize with Stylus

Customizing the block grid is easy with the Stylus variables provided in thesettings/block-grids.stylfile.

Stylus

include-block-grids = true

// Block Grids
block-grid-elements = 12
block-grid-default-space = rem-calc(20)

Semantic Markup with Stylus Mixins

You apply block grid styles to semantic markup using Stylus mixins.

Basic

You can use thecreate-block-grid()mixin to create your own block grid, like so:

Stylus

ul.custom-block-grid
  create-block-grid(20px, 3)

Advanced

You can further customize your block grid using the provided options in thecreate-block-grid()mixin:

.custom-block-grid
  create-block-grid(
    // This controls how much space is between each item in the block grid. Use a variable or any pixel or em value
    space: block-grid-default-space,
    // This controls how many elements will be on each row of the block grid for small device
    small: 1,
    // This controls how many elements will be on each row of the block grid for medium device
    medium: 1,
    // This controls how many elements will be on each row of the block grid for large device
    large: 1
  )

Stylus Errors?

Make sure you import this line in main file:
